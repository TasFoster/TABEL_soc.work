# Функция «Отчёт по госзаданию»

Формирует **отчёт о выполнении государственного задания** по одному соцработнику за
месяц в формате `.ods` (две вкладки: основные услуги и дополнительные). Запись `.ods` —
на чистом Python через `odfpy` (с нуля), Microsoft Excel не требуется.

- **Пакет:** `app/features/gos_zadanie/`
- **Ключ функции:** `gos_zadanie`
- **Вход:** один `.xls` — «Отчёт по количеству оказанных услуг и обслуженных граждан в
  разбивке» (выгрузка по одному соцработнику за месяц).
- **Выход:** `.ods` с **2 листами**: `Лист1` (госзадание — основные услуги) и
  `дополнительные`.

---

## Поток данных

```
источник .xls ──► parser.parse_source() ──► service.prepare()
                                                  │  (категоризация услуг по справочнику)
                                                  ▼
                                       writer.generate() ──► отчёт .ods (2 листа)
```

1. Пользователь выбирает файл-источник и нажимает «Подготовить».
2. `parser` читает по заголовкам реквизиты (соцработник, отделение, период) и таблицу
   получателей со счётчиками услуг.
3. `service.prepare` разносит услуги по категориям через **справочник** (`main`/`dop`/`skip`);
   неизвестная услуга по умолчанию идёт в `dop` и **дописывается в справочник**.
4. Реквизиты подставляются в окно (правятся); ФИО соцработника подбирается из БД по фамилии.
5. `writer.generate` строит `.ods`: на каждом листе заголовок, шапка, строки получателей,
   строка итогов, подписи.

---

## Слои

### GUI (`gui.py`) — `GosZadanieWindow`
Выбор источника, «Подготовить» → автозаполнение реквизитов (соцработник — combobox из БД,
отделение/зав./месяц/год — правятся), «Сформировать (.ods)»; кнопка отзывов; запись в архив
документов; память папок (`ui_state`).

### Сервис (`service.py`)
- `prepare(source_path)` — разбор + категоризация услуг по справочнику (+ авто-дозапись новых).
- `format_sign(fio)` — «Фамилия Имя Отчество» → «И.О.Фамилия» для подписи.
- `generate(prepared, out, worker, dept, month, year, zav)` — контекст + период за месяц
  (1-е … последнее число) → `writer`.

### Модель (`model.py`)
- `normalize(s)` — канонизация названия услуги для сравнения.
- `Client` — получатель (ФИО, дата рождения, пол, «Всего услуг», счётчики по услугам).
- `ServiceCatalog` — справочник: название → категория (`main`/`dop`/`skip`) + порядок;
  `lookup`/`category_of`/`order_of`/`add`/`to_list`.

### Парсер (`parser.py`)
Читает `.xls` (`xlrd`): ищет строку заголовков (есть «ФИО» и «Всего услуг»), размечает
служебные колонки (№/ФИО/дата рождения/пол/Всего услуг) и колонки услуг; реквизиты
(соцработник/отделение/период) — из верхних строк регулярками.

### Запись `.ods` (`writer.py`)
`odfpy` **с нуля** (`OpenDocumentSpreadsheet`): именованные стили (заголовок/шапка/ячейки/
итоги/подпись), ширины колонок, объединённый по колонкам заголовок (`numbercolumnsspanned`
+ `CoveredTableCell`). Итого: на листе госзадания = «Всего услуг» источника, на доп. листе =
сумма доп. услуг строки. Доп. лист — только получатели с доп. услугами.

### Хранилище (`storage.py`)
`load_services`/`save_services` — справочник `services_seed.json` (на первом запуске
копируется из ресурсов в `data/gos_zadanie/`); `soc_worker_fios` — соцработники из общей БД.

---

## Справочник услуг

`data/gos_zadanie/services_seed.json` — список `{name, category, order}`. Сопоставление
с услугой источника — по **нормализованному названию** (точное совпадение). Категории:
`main` (лист госзадания), `dop` (лист дополнительных), `skip` (не выводить). Колонки обоих
листов строятся из услуг, **реально присутствующих в источнике**, в порядке справочника, —
поэтому отчёт устойчив к смене набора услуг от месяца к месяцу.

## Проверка (self-test)

```
python main.py --selftest-gos <путь к файлу-источнику .xls>
```

Пишет `selftest_gos.ods` и `selftest_gos_result.txt` (строка `OK …` или трасса). Образец
формата — `Отчет соц раб/Отчет Бойкова.ods`; источник — `Отчет соц раб/Отчет по количеству
оказанных услуг … .xls`.
