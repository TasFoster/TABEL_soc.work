# Функция «Табель»

Формирует **табель учёта рабочего времени** по унифицированной форме **Т-13** (ОКУД 301008)
в формате `.xls` (Excel 97-2003). Запись `.xls` — на чистом Python через `xlwt`, форматирование
берётся из шаблона через `xlrd`. Microsoft Excel не требуется. Архитектура — MVC
([АРХИТЕКТУРА.md](../АРХИТЕКТУРА.md)): View=`gui.py`, Controller=`service.py`,
Model=`timesheet.py`/`calendar_ru.py`/`excel_writer.py`/`storage.py`.

- **Пакет:** `app/features/timesheet/`
- **Ключ функции:** `timesheet`
- **Шаблон:** `timesheet/templates/t13_template.xls`
- **Выход:** `Табель_<отделение>_<месяц>_<год>[_период].xls`

---

## Поток данных

```
GUI (отделение, месяц, год, ПЕРИОД, ввод отсутствий)
  └► service.generate_timesheet(..., period)
        ├► storage (отделение, настройки, календарь из app.db)
        ├► timesheet.build_marks (отметки по дням периода на каждого сотрудника)
        ├► calendar_ru.ProductionCalendar (выходные/праздники/часы; work_days)
        └► excel_writer.generate ──► .xls
```

---

## Период: месяц или полмесяца

В окне есть переключатель **«Период»**: *весь месяц* / *1–15* / *16–конец*.

- Форма Т-13 **та же**; при выборе половины заполняются **только дни периода**, остальные
  ячейки пустые; итоги (дни/часы) и неявки считаются **за период**.
- Имя файла получает суффикс `_1-15` / `_16-конец`.
- Реализация: `gui._selected_period(ndays)` → `None | (1,15) | (16,ndays)`;
  `service.build_context(..., period)` и `generate_timesheet(..., period)`;
  `timesheet.build_marks(year, month, cal, absences, day_from, day_to)` ограничивает расчёт
  днями периода.

## Слои

### View (`gui.py`) — `TimesheetWindow`

- Combobox **отделения**, **месяца** (`MONTHS_NOM`), Spinbox **года**; переключатель **периода**.
- Предупреждение, если год отсутствует в производственном календаре.
- Treeview сотрудников с колонкой **отсутствий**; добавление/очистка отсутствий
  (`AbsenceDialog`: день начала, день конца, код).
- Кнопки: **«Реквизиты и подписи…»** (`SettingsDialog`), **«Производственный календарь…»**
  (`CalendarDialog` — вкл. поле «Перенесённые РАБОЧИЕ дни»), **«Отделения и сотрудники…»**
  (`DepartmentManager`), **«Сформировать табель»** (`_generate`).
- `_generate` валидирует отсутствия (`validate_absences`), вычисляет период, предлагает имя
  файла, сохраняет и опционально открывает результат.

### Controller (`service.py`)

- `generate_timesheet(dept, year, month, absences_by_emp, out_path, settings, keep_open, period)` —
  строит контекст и вызывает писатель.
- `build_context(..., period=None)` — нерабочие дни и отметки считаются по периоду; на каждого
  сотрудника вызывает `build_marks`, собирает реквизиты/подписи.
- `get_calendar()` — `ProductionCalendar` с часами рабочего/предпраздничного дня из настроек.

### Model — данные/домен/запись

**`storage.py`** — обёртка над `core.db`: `load/save_departments`, `load/save_settings`,
`load/save_calendar`, `template_path`. Данные живут в общей `app.db`.

**`timesheet.py`** (домен):
- `build_marks(year, month, cal, absences, day_from=1, day_to=None)` →
  `{ndays, marks, worked_days, worked_hours, reasons}`. `marks[day]` = часы (8 или 7 в
  предпраздничный), `"В"` (выходной/праздник) или **код отсутствия**. Отметки/итоги/неявки —
  только за `day_from..day_to`; дни вне периода в `marks` не попадают.
- `validate_absences(...)` — проверка кода и диапазона `1..ndays`, `start ≤ end`.
- Коды отсутствий из `settings.json` (`absence_codes`, флаг `counts_as_absence`).

**`calendar_ru.py`** (домен):
- `ProductionCalendar(data, full_hours=8, short_hours=7)`; формат
  `{год: {holidays, short_days, work_days}}`.
- `is_holiday`, `is_short`, `is_weekend` (сб/вс), `is_extra_workday` (перенесённый рабочий
  день), `is_workday` (учитывает `work_days`), `hours(d)`.
- `work_days` правится в «Производственный календарь…»; используется и функцией «Проезд».

**`excel_writer.py`** (запись `.xls`):
1. Читает шаблон `xlrd.open_workbook(..., formatting_info=True)` (лист «Лист1»).
2. Транслирует XF-стили шаблона в `xlwt.XFStyle` (`core.xls_util.TemplateStyles`).
3. Копирует заголовок 1:1, реплицирует пару строк сотрудника N раз, сдвигает подписи на
   `2*(N-1)`; пересоздаёт объединения/высоты/ширины.
4. Заполняет ячейки; область дней очищается, затем пишутся только дни из `marks`.

**Раскладка (0-based):** день 1 → колонка **E** (`COL_DAY_FIRST=4`); дни 1–15 / 16–31 —
верх/низ (`day_cell`); итоги (дни/часы) → колонка **U** (`C_U=20`); причины — `REASON_COLS`;
подписи — строки 22–26 (+`shift`). Выходные/`"В"` заливаются синим `SHADE_RGB=(51,51,153)`
через `core.xls_util.ColourPalette`. Вывод — альбомная ориентация.

## Проверка (self-test)

```
python main.py --selftest        # построит табель (весь месяц) → selftest_result.txt
```

Эталон: корневой `табель май.xls` (значения + заливки + объединения + шрифты совпадают).
Половину месяца проверяют вручную в окне (переключатель «Период»); сумма половин (дни/часы)
равна полному месяцу.
