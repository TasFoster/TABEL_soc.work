# ГИД по директории «Мама работа»

Путеводитель по всей папке проекта: где что лежит и зачем. Подробная документация
кода — в `tabel_app/docs/` (начните с [`ОБЗОР.md`](tabel_app/docs/ОБЗОР.md) и
[`АРХИТЕКТУРА.md`](tabel_app/docs/АРХИТЕКТУРА.md)); правила для ИИ-ассистента — в
[`CLAUDE.md`](CLAUDE.md).

## Что это за проект

Настольное Windows-приложение (Python + Tkinter) для отдела соцобслуживания: из одного
главного меню открываются независимые **функции**, каждая формирует готовый
Excel/ODS/ODT-файл (табель, реестры, приложение, проезд, услуги-деньги, графики,
госзадание, проверка качества, пересмотр, планы). Офисный пакет (Word/Excel) не нужен —
все файлы пишутся на чистом Python.

## Карта директории

### Код и документация
| Папка/файл | Что это |
|---|---|
| **`tabel_app/`** | **Исходный код приложения.** Точка входа `main.py`, код в `app/`, документация в `docs/`, сборка `build.ps1`, версия `VERSION`. Единственное, что нужно для запуска/сборки. |
| `tabel_app/docs/` | Справочная документация: `ОБЗОР.md`, `АРХИТЕКТУРА.md` (опорный), `БАЗА_ДАННЫХ.md`, `функции/*.md` (по одному файлу на функцию). |
| `tabel_app/tools/` | Разовые вспомогательные скрипты (напр. `parse_plany_doc.py` — разбор исходных `.doc` планов в JSON-шаблон). |
| `CLAUDE.md` | Инструкции для ИИ-ассистента (Claude Code). |
| `README.md` | Как добавить новую функцию. |
| `ВОПРОСЫ.md` | Вопросы/решения, накопленные в ходе доработки. |
| `neuro/` | Отдельный эксперимент (нейросеть-OCR для номеров билетов). **В приложение не встроен.** |

| Папка | Для какой функции / что внутри |

| **`Образцы/`** | Разложенные по смыслу свободные файлы: `Табель/` (образцы табеля), `Приложение/`, `Журналы_договоров/` (журнал договоров + отчёт о заключённых), `_прочее/`, `_архив/` (старая Excel-версия `Табель.exe`, zip исходников планов). |

| `Проезд/` | «Проезд»: образцы сканов билетов и `.ods`-выходы. |

| `Пересмотр/` | «Пересмотр»: отчёт ИПСУ `…417.xls`. |

| `Отчет соц раб/` | Образцы для «Отчёта по госзаданию» («…в разбивке» `.xls`). |
| `протокол/` | Рабочие документы пользователя (методический час, протоколы) — не привязаны к конкретной функции. |

## Как запускать и собирать (кратко)

```
cd tabel_app
python main.py                                          # запустить приложение
powershell -ExecutionPolicy Bypass -File build.ps1                 # собрать полную .exe
powershell -ExecutionPolicy Bypass -File build.ps1 -Variant lite   # собрать лёгкую .exe
```

Самопроверки функций — `python main.py --selftest*` (список в `CLAUDE.md`). Подробности
сборки и поставки — в `tabel_app/docs/ОБЗОР.md` и `АРХИТЕКТУРА.md`.
