# Функция «Проверка качества»

Формирует лист **«Проверка качества социального обслуживания»** — план контроля работы
соцработников заведующей отделением: по четвергам месяца проверяются соцработники и
несколько их клиентов (ФИО, адрес, телефон, результат контроля). Формат — `.ods`,
запись на чистом Python через `odfpy` (с нуля, landscape), Microsoft Excel не требуется.

- **Пакет:** `app/features/proverka_kachestva/`
- **Ключ функции:** `proverka_kachestva`
- **Вход:** один `.xls` — **реестр** по оплате за оказанные социальные услуги (тот же
  формат, что у функции «Реестр»; парсер переиспользуется — `reestr.parser`).
- **Выход:** `.ods` с одним листом «Проверка качества» (альбомная ориентация).

Столбцы листа: **Дата | Ф.И.О. Социального работника | Ф.И.О. обслуживаемого | Адрес
проживания | Телефон | Результат контроля,наличие жалоб**. Объединённый заголовок и
подпись заведующей — редактируемые.

---

## Поток данных

```
реестр .xls ──► service.parse_workers_clients()  ─► {соцработник: [(клиент, адрес), …]}
                     (reestr.parser.parse_registry, 'gos')
                                   │
     выбор в окне: соцработник → четверг → клиенты (галочки) → строки превью
                                   │
                       service.generate() ─► writer.generate() ─► лист .ods
```

1. Пользователь выбирает файл реестра и нажимает «Загрузить». Соцработники и их клиенты с
   адресами извлекаются из реестра (ФИО клиента — без даты рождения; двойное ФИО
   соцработника «…, …» — совместное обслуживание — разбивается, клиент попадает к обоим).
2. Заведующая выбирает соцработника, один из **четвергов месяца** (считаются автоматически)
   и отмечает галочками его клиентов (без ограничения по числу) → строки добавляются в
   таблицу-превью с автоподстановкой сохранённого телефона и результатом «нет».
3. Любую ячейку таблицы можно отредактировать двойным кликом; строки можно добавлять и
   удалять. Перед формированием строки сортируются по дате.
4. `writer.generate` строит `.ods`: объединённый заголовок, шапка, строки, подпись.

## Слои

### GUI (`gui.py`) — `ProverkaKachestvaWindow`
Период (месяц/год), реквизиты (отд. №, зав. отделением), редактируемые заголовок и
подпись, выбор файла реестра, блок назначения (соцработник → четверг → клиенты-галочки),
таблица-превью (`ttk.Treeview`) с инлайн-редактированием ячеек по двойному клику и
кнопками добавления/удаления строк; кнопка отзывов; запись в архив документов; память
папок и последних отд./зав. (`ui_state`). Заголовок/подпись пересобираются из реквизитов,
пока пользователь не изменил их вручную.

### Сервис (`service.py`)
- `month_thursdays(year, month)` — все четверги месяца (`weekday()==3`).
- `format_date(d)` — дата в формате ДД.ММ.ГГ.
- `split_workers(raw)` — «Кошелева С.А., Смарунь Т.В.» → два ФИО (совместное обслуживание).
- `parse_workers_clients(path)` — реестр → `{order, by_worker}` (переиспользует
  `reestr.parser.parse_registry(path, 'gos')`; дедупликация клиентов у соцработника).
- `default_title(dept_no)` / `default_sign(dept_no, zav)` — тексты заголовка и подписи.
- `generate(out, ctx, rows)` — сортировка строк по дате → `writer`.

### Запись `.ods` (`writer.py`)
`odfpy` **с нуля** (`OpenDocumentSpreadsheet`), приёмы как в `gos_zadanie/writer.py`:
именованные стили (заголовок 16pt italic bold, шапка, ячейки Arial 11pt с рамкой,
подпись), ширины колонок, объединения (`numbercolumnsspanned` + `CoveredTableCell`).
**Альбомная ориентация** задаётся страничным стилем (`PageLayout` + `MasterPage` +
table-style с `masterpagename`). Даты пишутся строковыми ячейками, чтобы ДД.ММ.ГГ не
превратилось в число.

### Хранилище (`storage.py`)
Телефоны клиентов поверх общей БД: `load_phone(fio)`, `save_phone(fio, phone)`,
`load_all_phones()`. В реестре телефонов нет — их вводят вручную в окне и запоминают по
ФИО клиента (таблица `pk_phones`), чтобы подставлять в следующий раз.

## Проверка (self-test)

```
python main.py --selftest-pk <путь к реестру .xls>
```

Пишет `selftest_pk.ods` и `selftest_pk_result.txt` (строка `OK …` или трасса) — демо-лист
по первым соцработникам и клиентам, разложенным по четвергам текущего месяца. Образец
формата — `Проверка качества/проверка качества 2025 июль.ods`; вход —
`Проверка качества/Реестр по оплате за оказанные социальные услуги_242.xls`.
