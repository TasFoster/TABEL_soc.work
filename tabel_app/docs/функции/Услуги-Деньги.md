# Функция «Услуги-Деньги»

Формирует отчёт **«Услуги-Деньги»** (`.xlsx`, 3 листа) из сводки количества услуг и
списков льготников. Заполняет **шаблон с готовыми формулами и тарифами** через `openpyxl`
(Microsoft Excel в работе не нужен). Архитектура — MVC ([АРХИТЕКТУРА.md](../АРХИТЕКТУРА.md)).

- **Пакет:** `app/features/uslugi_dengi/`
- **Ключ функции:** `uslugi_dengi`
- **Входы:** отчёт **071** (`.xls` — количество услуг), **бесплатники-частичники**
  (`.xlsx` **или `.xls`**) и (необязательно) **отчёт за прошлый месяц** (`.xlsx/.xls`)
  для накопления.
- **Выход:** `.xlsx` из 3 листов: **Услуги-Деньги**, **Бесплатники**, **Частичники**.

---

## Поток данных

```
071.xls ──► parser.read_071 ─┐
                             ├─ service.prepare → агрегаты по услугам ─► writer.generate
бесплатники-частичники.xlsx ─┘   (canon: всего/частично/бесплатно)        (заполнение шаблона)
                                                                              │
                                                              uslugi_dengi_template.xlsx → отчёт.xlsx
```

1. Из **071** берётся количество каждой услуги (всего).
2. Из **бесплатники** — кол-во по услугам у бесплатников; из **частичники** — кол-во и
   деньги (два блока в файле).
3. На листе **Услуги-Деньги** программа заполняет только **вводные** ячейки, а **формулы**
   и **тарифы** уже в шаблоне и считают сами:

| Кол. | Поле | Тип | Источник |
|---|---|---|---|
| C | Количество услуг (всего) | ввод | 071 |
| E | Стоимость по тарифу | в шаблоне | — |
| H | Кол-во частичная | ввод | частичники (кол-во) |
| I | Оплачено частичная | ввод | частичники (деньги) |
| J | Кол-во бесплатно | ввод | бесплатники (кол-во) |
| F | Кол-во полная оплата | формула `=C−J−H` | шаблон |
| G | Оплачено полная | формула `=F×E` | шаблон |
| D | Оплачено всего | формула `=G+I` | шаблон |

4. Листы **Бесплатники/Частичники** заполняются клиентами (ФИО + кол-во по услугам), с
   построчным «Итого» и итоговыми суммами; у частичников два блока — кол-во и деньги.

## Накопление (отчёт нарастающим итогом за год)

Столбцы **H/I/J** на листе «Услуги-Деньги» — **накопительные за год**: значение текущего
месяца = **значение прошлого месяца + новое** (новое берётся из бесплатники-частичники за
текущий месяц). Поэтому при формировании указывается **отчёт за прошлый месяц** как основа:

- **вручную** — поле «Отчёт за прошлый месяц» (можно указать прошлый `.xlsx` этой программы
  или старый ручной `.xls` — H/I/J читаются по названию услуги из колонок 8/9/10);
- **автоматически** — если поле пустое, берётся отчёт за предыдущий месяц того же года из
  **архива документов** (`service.find_prev_report`);
- в **январе** основы нет — накопление начинается заново (`month<=1` → без базы).

Лист 071 (**C**) не накапливается (пользователь подаёт 071 за нужный период); листы
клиентов — за текущий месяц. Реализация: `parser.read_prev_ud` (читает H/I/J предыдущего
отчёта), `writer._fill_ud` складывает базу с новыми значениями.

## Сопоставление услуг (canon)

Между файлами (071 / шаблон / xlsx) услуги сопоставляются **по названию** (номера колонок
несогласованы, есть опечатки), поэтому у каждой услуги — устойчивое ключевое слово
(`model.SERVICES`, `model.canon_of`). Для каждой услуги задано: строка на «Услуги-Деньги»
(`ud_row`) и колонка на листах клиентов (`client_col`). 12 услуг формы «на дому»
(продукты/промтовары/санитария/книги/приготовление/вода/оплата/уборка/отправка/документы/
психолог/обучение).

## Слои

- **`model.py`** — канон услуг + `canon_of(name)` (сопоставление по ключевому слову).
- **`parser.py`** — `read_071` (xlrd: услуга→количество), `read_benefit` (клиенты и агрегаты
  бесплатников/частичников; у частичников блоки кол-ва и денег) и `read_prev_ud` (H/I/J
  предыдущего отчёта по названию услуги). Принимает **и `.xlsx`, и `.xls`**: `_load_book`
  отдаёт openpyxl для `.xlsx` либо xlrd-адаптер (`_XlsBook`/`_XlsSheet`/`_XlsCell` с
  openpyxl-совместимым 1-based `cell()`/`max_row`/`max_column`/`title`) для `.xls`.
- **`service.py`** (Controller) — `prepare(071, xlsx, prev_path=None)` → данные (+`prev_ud`);
  `find_prev_report(месяц, год)` (поиск прошлого месяца в архиве); `generate(data, out,
  месяц, год, отд.)`; `period_text` («За {месяц} {год} года в отделении №N»).
- **`writer.py`** (Model/IO) — `openpyxl` загружает шаблон (формулы сохраняются), заполняет
  C и **накопительные** H/I/J (база+новое) на «Услуги-Деньги» и листы клиентов; объединённые
  ячейки листов клиентов разъединяются перед записью.
- **`storage.py`** — путь к шаблону `uslugi_dengi_template.xlsx`.
- **`gui.py`** (View) — выбор 071, бесплатники-частичники и (необязательно) прошлого отчёта
  + месяц/год/отделение + «Сформировать».

---

## Шаблон

`templates/uslugi_dengi_template.xlsx` — сделан из образца (`услуги деньги/…печать.xls`),
**формулы (D/F/G) и тарифы (E) сохранены**, данные очищены. Образец `.xls` разово
сконвертирован в `.xlsx` через Excel COM (только для подготовки шаблона; в приложении Excel
не нужен — `openpyxl` читает готовый `.xlsx`).

## Проверка (self-test)

```
python main.py --selftest-uslugi <папка с 071.xls и бесплатники-частичники.xlsx>
```

Папка `услуги деньги/` в корне — готовый набор входов. Проверено: 12 услуг из 071,
23 бесплатника, агрегаты частичников; формулы D/F/G в выходе сохранены.
