# Функция «Пересмотр»

Показывает получателей, у которых **заканчивается срок обслуживания** в заданном
месяце/году — из отчёта **ИПСУ**. Результат — список в окне (ФИО + дата окончания)
с сохранением в `.ods`. Запись `.ods` — на чистом Python через `odfpy` (с нуля).

- **Пакет:** `app/features/peresmotr/`
- **Ключ функции:** `peresmotr`
- **Вход:** один `.xls` — «Отчёт по формированию реестра ИПСУ» (тот же формат, что
  использует лист «пересмотр» функции «Реестр»; разбор переиспользуется —
  `reestr.parser.parse_ipsu`).
- **Выход:** `.ods` — список (2 колонки: ФИО, дата окончания срока).

---

## Логика

- Срок берётся из колонки **«Срок предоставления услуги»** (диапазон
  «дд.мм.гггг - дд.мм.гггг»); **окончанием** считается ПОСЛЕДНЯЯ дата диапазона
  (`service.end_date`).
- В список попадают записи, чья дата окончания приходится **ровно на заданный
  месяц и год**. Строки выводятся **как есть** — если у человека несколько строк
  ИПСУ, показываются все.
- Период задаётся в окне (месяц + год, по умолчанию текущие).

## Слои

- **`gui.py`** — `PeresmotrWindow`: выбор месяца/года, выбор файла ИПСУ, кнопка
  «Найти» → таблица (`ttk.Treeview`), кнопка «Сохранить (.ods)»; кнопка отзывов,
  архив документов, память папок (`ui_state`).
- **`service.py`** — `month_thursdays`-подобной логики нет; ключевое:
  `end_date(srok)`, `find_expiring(path, year, month)` (переиспользует
  `reestr.parser.parse_ipsu`), `default_title(year, month)`, `generate`.
- **`writer.py`** — `.ods` с нуля через `odfpy`: объединённый заголовок, шапка,
  строки (ФИО | дата окончания). Приёмы как в `gos_zadanie`/`proverka_kachestva`.

Своих таблиц БД функция не создаёт (только читает входной `.xls`).

## Проверка (self-test)

```
python main.py --selftest-peresmotr <путь к отчёту ИПСУ .xls>
```

Пишет `selftest_peresmotr.ods` и `selftest_peresmotr_result.txt` (`OK …` или трасса).
Образец входа — `Пересмотр/Отчет по формированию реестра ИППСУ 417.xls`.
