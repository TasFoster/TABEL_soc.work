# Приложение «Рабочие функции» (модульное)

Десктоп-приложение (Windows) с **главным меню функций**. Сейчас доступна одна
функция — «Табель учёта рабочего времени» (форма Т-13). Архитектура рассчитана
на простое добавление новых функций.

## Структура

```
tabel_app/
  main.py                       — точка входа: запускает главное меню
                                   (скрытый режим --selftest для проверки .exe)
  app/
    __init__.py
    shell.py                    — главное меню (лаунчер функций)
    core/                       — ОБЩИЙ слой (не зависит от функций)
      storage.py                — хранение JSON по функциям (пути для .exe/исходников)
      db.py                     — ОБЩАЯ база SQLite (data/app.db) — единый источник
                                  для всех функций: отделения, сотрудники, реквизиты,
                                  календарь, таблицы «Приложения» + клиенты «Реестра»
                                  и их группы (reestr_*). Табель и Реестр читают/пишут
                                  эту же базу. При первом запуске засевается из JSON,
                                  далее правки идут в базу (JSON — резервные дефолты).
    features/
      __init__.py
      registry.py               — СПИСОК функций (здесь регистрируются новые)
      timesheet/                — функция «Табель» (всё своё — внутри)
        __init__.py             — ключ/название/описание функции
        gui.py                  — окно функции (open_timesheet / TimesheetWindow)
        service.py              — сборка контекста (данные + календарь + расчёт)
        timesheet.py            — расчёт отметок, итогов, «неявок по причинам»
        calendar_ru.py          — производственный календарь (тип дня, часы)
        excel_writer.py         — запись в .xls на чистом Python (xlwt + xlrd):
                                  переносит оформление шаблона и размножает строки
        storage.py              — доступ к общей БД (core.db): отделения/сотрудники,
                                  настройки, календарь
        data/                   — значения по умолчанию (departments/settings/calendar)
        templates/              — шаблон t13_template.xls
      reestr/                   — функция «Реестр по оплате за услуги» (.ods)
      prilozhenie/              — функция «Приложение к табелю» (чел/дни, .xls)
        model.py                — Worker/Period/Absence/Redistribution, порог 20/16
        calc.py                 — чистый расчёт: сетка по дням, обнуление при
                                  отсутствии, ручное перераспределение, итоги,
                                  начисление (оклад/(норма×раб.дни)×факт), предупр.
        storage.py              — доступ к общей БД (core.db) для этой функции
        service.py              — календарь + compute + запись .xls
        excel_writer.py         — лист строится программно через xlwt
                                  (левый «Расчёт» по дням + правый «Приложение»)
        gui.py                  — окно (нагрузка/периоды/отсутствия/перераспред.)
        data/                   — loads_seed.json (стартовая нагрузка по ФИО)
```

В собранном `.exe` ресурсы функций зашиты внутрь (под `features/<имя>/...`);
при первом запуске данные копируются в `data/<имя>/` рядом с `.exe` и далее
редактируются пользователем.

## Как добавить НОВУЮ функцию

1. Создайте папку `app/features/<имя>/` с `__init__.py`
   (определите `FEATURE_KEY`, `FEATURE_TITLE`, `FEATURE_DESCRIPTION`).
2. Сделайте окно функции — класс на базе `tk.Toplevel` и функцию
   `open_<имя>(master) -> Toplevel`.
3. Если нужны свои данные/шаблоны — положите их в `app/features/<имя>/data`
   и `.../templates`, а для доступа сделайте тонкую обёртку по образцу
   `timesheet/storage.py` (она фиксирует ключ функции).
4. Добавьте одну строку `Feature(...)` в `app/features/registry.py`.
5. В `build.ps1` добавьте `--add-data` для ресурсов новой функции
   (назначение — `features/<имя>/...`).

Главное меню само покажет новую кнопку.

## Запуск из исходников

```
python main.py
```

## Зависимости

Python 3.x, Windows. **Microsoft Excel НЕ требуется** — файлы пишутся на чистом
Python: `.xls` через `xlwt`, `.ods` через `odfpy`; чтение исходных `.xls`
(реестры) и шаблона табеля — через `xlrd`.

```
pip install -r requirements.txt   # xlrd, xlwt, odfpy
```

## Логика «Табеля» (проверено на заполненном образце)

- Рабочий день → часы (8, либо 7 в предпраздничный день из календаря).
- Выходной/праздник/перенос → «В» (ячейка заливается RGB 51,51,153).
- Период отсутствия (Б/ОТ/ОЖ и др.) перекрывает все календарные дни периода,
  включая выходные внутри; такие дни не заливаются и не считаются отработанными.
- Итог: «дни»/«часы» в колонке U (верх/низ).
- «Неявки по причинам»: код + дни; код в той половине месяца, где начинается
  отсутствие (1-я → верхняя строка, 2-я → нижняя).
- Подсветка выходных в шапке и данных — по выходным/праздникам ТЕКУЩЕГО месяца.

Сгенерированный табель совпал с заполненным человеком образцом (Отд №2, май 2026)
по значениям и по цвету заливки — расхождений нет.

## Сборка .exe

```
powershell -ExecutionPolicy Bypass -File build.ps1
```
Сборка идёт в ASCII-папке (путь проекта содержит кириллицу). Результат —
`Tabel.exe` (onefile, ~13 МБ); переименуйте в `Табель.exe` и положите рядом с
будущей папкой `data` (создаётся при первом запуске).
