# Функция «Реестр по оплате»

**Правит готовый реестр `.ods`** (листы Гос_/Доп/деньги с живыми формулами) по
**«Отчёту по количеству заключённых договоров»** за месяц: обновляет суммы,
помечает/добавляет новых и обнуляет снятых клиентов, а затем **пересобирает все
живые формулы** (Итого/Всего/деньги/Доп) с правильными адресами по итоговой
раскладке. Microsoft Excel не требуется. Заменяет прежнюю функцию «Реестр» в меню.

- **Пакет:** `app/features/reestr_oplata/`
- **Ключ функции:** `reestr_oplata`
- **Вход:** готовый реестр `.ods` (Гос_/Доп/деньги) + журнал `.xls` («Отчёт по
  количеству заключённых договоров»).
- **Выход:** тот же реестр `.ods` с обновлёнными суммами/строками и пересобранными
  формулами (по умолчанию сохраняется под тем же именем).

---

## Логика

Сопоставление клиентов — **по ФИО (+ № договора)**, как во всём реестре (номер
строки между месяцами нестабилен). Журнал разбирается `reestr.journal.
parse_journal_detailed` — плоский список договоров за месяц с **суммой за отчётный
месяц** по каждому (гос и доп раздельно) и периодом.

Категории изменений плана (`service.compute_plan`), подтверждаемые галочками:

- **Суммы (Гос)** — у существующего клиента изменились E/F (к оплате) и/или H (доп).
- **Снятые (Гос)** — клиента нет в журнале → **обнулить** суммы, строку оставить.
- **Новые клиенты (Гос)** — есть в журнале, нет в реестре → добавить строку; в
  журнале **нет соцработника**, поэтому его назначают вручную (двойной клик).
- **Доп: обновить / новые / снятые** — синхронизация листа Доп с журналом.
  Ложные пары «снять доп + новая доп» одного ФИО (когда «Доп. соглашение №N к
  договору …» не совпадает по номеру) схлопываются (`_collapse_dop_relabels`).

Пересборка формул (`ods_editor.rebuild_formulas`) — **суть функции**: диапазоны
`SUM(...)` в строках Итого блоков, суммы ячеек Итого в «Всего», ссылки листа
«деньги» на `[Гос_.I…]`, ИТОГО листа Доп — все адреса вычисляются по **финальной**
раскладке (1-based, с учётом добавленных строк). Кэш-значения проставляются для
показа до пересчёта. Обновляются и письма «Предоставляем денежные средства в
объёме X (прописью) … в <месяце> <год> г.» (сумма прописью + месяц/год — важно при
правке реестра прошлого месяца новым журналом).

## Слои (MVC)

- **`model.py`** — датаклассы `GosClient/GosBlock/DopRow/DengiRow/RegistryModel`;
  каждая строка хранит **ссылку на реальный odfpy-элемент** (правки идут прямо в
  дерево, адреса формул пересобираются по факту).
- **`ods_editor.py`** (Model/IO) — `load_model` (устойчивый разбор «грязного»
  ручного реестра: якорь по маркерам «Социальный работник», строка Итого по
  SUM-формуле, а не по тексту метки), операции правки строк
  (`set_gos_sums/set_gos_dop/zero_gos_client/add_gos_client/set_dop_sum/
  add_dop_row/renumber_block`), `rebuild_formulas`, `save`.
- **`service.py`** (Controller) — `analyze` (реестр+журнал→модель+план),
  `compute_plan`, `apply_plan`, `worker_names`.
- **`gui.py`** (View) — `ReestrOplataWindow`: выбор двух файлов → «Проанализировать»
  → `ttk.Treeview`-чеклист (клик по «✔» — применять/нет; двойной клик по
  «Соцработник» у новых — комбобокс) → «Применить и сохранить». Кнопка отзывов,
  архив документов, память папок (`ui_state`).

Своих таблиц БД функция не создаёт (правит переданный `.ods`, читает `.xls`).
Готовый файл архивируется в «Сохранённые документы» (`documents`).

## Проверка (self-test)

```
python main.py --selftest-reestr-oplata <реестр.ods> <журнал.xls>
```

Пишет `selftest_reestr_oplata.ods` и `selftest_reestr_oplata_result.txt` (`OK …`
или трасса). Образцы входа — `Риестр/РЕЕСТР июнь №9.ods` и
`Риестр/Отчет по количеству заключенных договоров 381.xls`. Проверка согласованности
после применения: Гос_ «Всего» H = ИТОГО листа Доп; «деньги» ИТОГО = Гос_ «Всего» I.
