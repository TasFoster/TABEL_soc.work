# Функция «Протокол»

Формирует **протокол рабочего совещания (методчаса)** отделения в формате `.odt`
(через `odfpy`, без Word). Архитектура — MVC ([АРХИТЕКТУРА.md](../АРХИТЕКТУРА.md)).

- **Пакет:** `app/features/protokol/`
- **Ключ функции:** `protokol`
- **Вход:** реквизиты в окне (№ протокола, дата совещания, отделение, зав.) +
  присутствующие из общей базы (соцработники отделения, галочки) + редактируемый
  текст повестки/решений.
- **Выход:** `.odt` — протокол по вёрстке образца (`протокол/Протоколы/…`).

---

## Что генерируется (по образцу `протокол/Протоколы/отд №9/…`)

- «**Протокол № N**» — по центру, жирный, Times New Roman 14pt.
- «рабочего совещания отделения социального обслуживания на дому № {отд} от {дата}» —
  по центру.
- «**Присутствовали:**» + список ФИО; затем «**Отсутствовали:**» + список снятых галочкой
  (если есть).
- «**Повестка дня:**», пункты повестки **из плана методчаса за выбранный месяц**, по каждому
  содержательному вопросу «По N-му вопросу заслушали зав. отд. …» + «Решение: …», блок «**Разное:**».
- Подпись «Зав.отделением №{отд}                {Фамилия И.О.}».

---

## Главное правило (правки по отзыву пользователя 2026-06-22)

От протокола к протоколу меняются № протокола, **месяц/год**, отделение и состав. Ключевое:
- **Дата ставится автоматически** — последняя **рабочая среда** выбранного месяца
  (методический час; `service.last_working_wednesday`, по производственному календарю; поле
  редактируемо).
- **Повестка/«кого заслушали» берутся из плана методического часа** за выбранный месяц
  (`storage.load_plan()` ← `data/plan.json`; `service.default_body(month)` строит повестку +
  решения). Кнопка «Повестка из плана» перезаполняет текст; правится в окне.
- **Отсутствующие**: снятые галочки попадают в блок «Отсутствовали:».
Присутствующие берутся из той же базы, что и «График проверок», по умолчанию **отмечены все**.
План методчаса (`data/plan.json`, 12 месяцев) сидится из дефолта и правится пользователем;
исходный план — `протокол/Методический час/План…отд.№9.doc` (разобран разово через Word COM).

## Слои

- **`service.py`** (Controller) — `DEFAULTS` (№ отд., зав.), `DEFAULT_BODY` (текст
  повестки/решений), `default_body(zav)` (подстановка фамилии зав.),
  `format_date(date)`, `generate(out, number, date, attendees, body, dept_no, zav)`.
- **`writer.py`** (Model/IO) — `odfpy` строит `.odt` **с нуля**, воспроизводя вёрстку:
  именованные стили абзацев (заголовок по центру жирный / тело слева, Times New Roman
  14pt), повторяющиеся пробелы кодируются `<text:s>` (иначе ODT их схлопывает).
- **`storage.py`** — отделения и соцработники из общей базы (`list_departments`,
  `soc_workers` — те же, что у «Графика проверок»).
- **`gui.py`** (View) — выбор отделения, № протокола, дата, поля отд.№/зав.,
  список присутствующих с галочками, текстовое поле повестки/решений, «Сформировать».

---

## Сохранение

Сформированный файл, как и у всех функций, **сохраняется копией в базу** (архив
«Сохранённые документы», `core/documents.py`) — можно открыть/пересохранить позже.

## Проверка

`python main.py --selftest-protokol` — формирует протокол по соцработникам первого
отделения из базы → `selftest_protokol.odt` + `selftest_protokol_result.txt`.
Проверено: 16 присутствующих, вёрстка (центр/слева, Times New Roman 14, подпись с
отступом) совпадает с образцом, файл открывается в LibreOffice/Word.
